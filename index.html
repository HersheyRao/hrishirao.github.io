Artifact 1: League of Legends Challenger Data Science Tutorial

Link to project:
https://hersheyrao.github.io/

Contextual Statement

This artifact is a data science tutorial website I built for CMSC320 that walks through the full data science pipeline using the then current official League of Legends Challenger dataset. I pulled data from the Riot Games API for all 300 North American Challenger players, cleaned it into a pandas dataframe, and then walked through exploratory analysis, visualizations, and basic machine learning modeling. The site is written as a step by step notebook style guide so a reader can see the raw API pulls, the code to shape the data, and the final plots and statistics in one place.

This project shows that I can handle the technical side of data science and explain it in an understandable way. I had to work around API rate limits, design and test model ideas, and use pandas, NumPy, matplotlib, and seaborn to explore relationships like win rate versus league points and total games played. I also had to decide which results were meaningful instead of flooding the page with graphs. The write up also forced me to translate numbers into plain language so someone who cares about League of Legends but is not a data scientist can still follow along.

This artifact fits my portfolio because it combines something I actually care about, high level League of Legends play, with legitimate work in data science. It shows I can take an open ended question, build my own dataset, and carry it from collection to interpretation on my own. It also shows that I can present technical work in a public facing format, which is important for jobs where you have to communicate results to nontechnical audiences.

Artifact 2: Wild Wild Magic (Roguelite Dungeon Game)

Download links:
Windows build: Google Drive (Windows)
Mac build: Google Drive (Mac)

Installation Instructions

The title of the application is new1. By downloading the folder and launching the new1 executable, the game should run.

Contextual Statement

This artifact is a roguelite dungeon game my group built for CMSC425 using Unity 6 and C#. The player spawns in a maze, fights bats and golems, and picks an augment after clearing the room. We made almost everything using basic Unity shapes and textures we drew ourselves. The goal was to build a simple dungeon environment that still fit a fantasy theme. We also used ideas from class like tremble effects for the bats and simple audio manipulation for the player’s attacks and the environment.

The maze is built using procedural generation. The game places a starting room at the center, then randomly places additional rooms around it on a grid. Each prefab has openings on four sides, and the generator checks which entrances should remain open. If a room is placed at an outside edge, any unused openings are filled with walls so the maze remains consistent. The game also randomizes the number and placement of enemies. Some rooms may have only a couple of bats while others may contain multiple bats and golems. This keeps each run slightly different.

This artifact fits my portfolio because it shows I can work on a complete game loop and handle interaction between multiple systems: procedural generation, enemy behavior, movement, upgrades, and visuals. It required programming, teamwork, and attention to how the parts fit together. The game is simple, but it is fully playable and easy for someone to understand immediately.

Artifact 3: Fraud Compiler

Download full project:
CMSC430 compiler code (ZIP on Google Drive)

Contextual Statement

This artifact comes from my CMSC430 project, where we built a working compiler for a small functional language called Fraud. A compiler takes code written by a person and turns it into instructions a machine can actually run. Working on something like this makes you deal directly with how languages, data, and memory behave underneath the surface, since nothing hides the details for you.

My main role in the project was writing the part of the compiler that converts high level expressions such as if statements, let bindings, case expressions, functions, and function calls into a86 assembly. That involved generating labels, managing the stack, deciding how values move through registers, and making sure the compiled program follows the calling conventions the runtime expects. Even though the file is long, it is the center of the compiler because this is where a Fraud program actually becomes something the machine can execute.

Zooming out, this project matters because it forces you to think about how programming languages work at every level. Most programmers write code without thinking about how variables map to memory, how control flow jumps, or how function calls pass data around. Building a compiler makes all of that visible. For someone who is not technical, the simplest way to understand this project is to think of it as building the “translator” that lets humans talk to hardware.

I chose this artifact because it shows that I can work inside a large codebase, understand a custom language’s rules, and implement them reliably in assembly. It also pairs well with my other artifacts since it represents the low level, systems side of what I can do. It complements the data science work in Artifact 1 and the Unity engineering in Artifact 2.

#lang racket
(provide compile
         compile-e)

(require "ast.rkt")
(require "compile-ops.rkt")
(require "types.rkt")
(require a86/ast a86/registers)

;; ClosedExpr -> Asm
(define (compile e)
  (prog (Global 'entry)
        (Extern 'peek_byte)
        (Extern 'read_byte)
        (Extern 'write_byte)
        (Extern 'raise_error)
        (Label 'entry)
        (Mov r11 rbx)
        (Sub rsp (* 8 14))
        (Mov (Mem rsp 0)   r12)
        (Mov (Mem rsp 8)   r13)
        (Mov (Mem rsp 16)  r14)
        (Mov (Mem rsp 24)  r15)
        (Mov (Mem rsp 32)  rbx)
        (Mov (Mem rsp 40)  rbp)
        (Mov (Mem rsp 48)  rdi)
        (Mov (Mem rsp 56)  rsi)
        (Mov (Mem rsp 64)  r8)
        (Mov (Mem rsp 72)  r9)
        (Mov (Mem rsp 80)  r10)
        (Mov (Mem rsp 88)  r11)
        (Mov (Mem rsp 96)  rax)
        (Mov (Mem rsp 104) rcx)
        (Mov (Mem rsp 112) rdx)
        (Mov rbx r11)
        (compile-e e '())
        (Mov r12 (Mem rsp 0))
        (Mov r13 (Mem rsp 8))
        (Mov r14 (Mem rsp 16))
        (Mov r15 (Mem rsp 24))
        (Mov rbx (Mem rsp 32))
        (Mov rbp (Mem rsp 40))
        (Mov rdi (Mem rsp 48))
        (Mov rsi (Mem rsp 56))
        (Mov r8  (Mem rsp 64))
        (Mov r9  (Mem rsp 72))
        (Mov r10 (Mem rsp 80))
        (Mov r11 (Mem rsp 88))
        (Mov rax (Mem rsp 96))
        (Mov rcx (Mem rsp 104))
        (Mov rdx (Mem rsp 112))
        (Add rsp (* 8 14))
        (Ret)))

;; Expr CEnv -> Asm
(define (compile-e e cenv)
  (match e
    [(Lit d)
     (compile-datum d)]

    [(Prim0 p)
     (compile-op0 p)]

    [(Prim1 p e1)
     (seq (compile-e e1 cenv)
          (compile-op1 p))]

    [(Prim2 p e1 e2)
     (seq (compile-e e1 cenv)
          (Push rax)
          (compile-e e2 (cons #f cenv))
          (compile-op2 p))]

    [(Case e ds es el)
     (let* ([m (length ds)]
            [l-end  (gensym 'case_end)]
            [l-hits (build-list m (λ (_) (gensym 'case_hit)))])
       (seq
        (compile-e e cenv)
        (Push rax)
        (foldl
         (λ (i acc)
           (let* ([datums (list-ref ds i)]
                  [l-next (gensym 'case_next)])
             (seq acc
                  (foldl
                   (λ (d acc2)
                     (seq acc2
                          (Mov rax (Mem 0 rsp))
                          (Mov r9  (value->bits d))
                          (Cmp rax r9)
                          (Je (list-ref l-hits i))))
                   (seq)
                   datums)
                  (Jmp l-next)
                  (Label (list-ref l-hits i))
                  (compile-e (list-ref es i) cenv)
                  (Jmp l-end)
                  (Label l-next))))
         (seq)
         (range 0 m))
        (compile-e el cenv)
        (Label l-end)
        (Add rsp 8)))]

    [(Let x e1 e2)
     (let* ([n (length x)]
            [c* (append (reverse x) cenv)])
       (seq
        (for/fold ([acc (seq)]) ([ei e1])
          (seq acc
               (compile-e ei cenv)
               (Push rax)))
        (compile-e e2 c*)
        (Add rsp (* 8 n))))]

    [(Var x)
     (seq (Mov rax rsp)
          (Add rax (lookup x cenv))
          (Mov rax (Mem 0 rax)))]))
